# 廖总 Windows NapCat 一键安装指南

> 目标：让你的 Windows 电脑能"自动监听 QQ 群消息"，把每一条群消息实时发到我的电脑上。
> 操作时间：**5 分钟**。不需要写代码。

---

## 准备

- ✅ 一台 Windows 电脑（你现有的就行，64 位）
- ✅ **专门开一个 QQ 小号**（强烈推荐，**不要用主号**，QQ 安全中心可能会封号）
- ✅ 提前把要采集的 QQ 群拉这个小号进去

---

## 步骤 1：装 QQ（推荐版本，5 分钟）

NapCat 官方钦点的 QQ 版本是 **9.9.26.44343**。装其他版本可能不兼容。

下载链接：
👉 https://dldir1.qq.com/qqfile/qq/QQNT/40d6045a/QQ9.9.26.44343_x64.exe

下载后双击 `.exe` 直接装。**用刚才说的专用小号扫码登录**。

> 如果你 Windows 已经装了 QQ 但版本不对：先在控制面板卸载旧 QQ，再装这个版本。

---

## 步骤 2：装 NapCat（一键脚本，1 分钟）

下载 NapCat **一键安装包**（只有 1MB）：
👉 https://github.com/NapNeko/NapCatQQ/releases/download/v4.18.1/NapCat.Shell.Windows.OneKey.zip

> 如果 GitHub 打不开，让我帮你传到飞书云盘备用。

**操作**：
1. 把下载的 `NapCat.Shell.Windows.OneKey.zip` 解压到桌面（右键→解压到当前文件夹）
2. 解压出来的文件夹里有个 `NapCatInstaller.exe`，**右键 → 以管理员身份运行**
3. 脚本会自动检测你的 QQ 安装位置，自动下载完整 NapCat 包并装好
4. 装完会弹出一个"启动器"窗口，**不要关闭它**

> 如果 Windows Defender 拦截了，点"仍要运行"。NapCat 是开源软件，安全可用。

---

## 步骤 3：用专用小号登录 QQ（NapCat 模式，1 分钟）

1. NapCat 启动器窗口里点 **"启动 QQ"** 按钮（或者从开始菜单找 NapCat 启动器）
2. QQ 会弹出**扫码登录界面**
3. **用专用采集小号**扫码登录
4. 登录后 NapCat 启动器会显示 ✅ 已连接

---

## 步骤 4：配置 HTTP 上报地址（**关键，必须做**）

这一步告诉 NapCat："**每收到一条群消息，立刻发到这个网址**"。

### 4.1 打开 NapCat WebUI 控制台

NapCat 启动器里有个**"打开 WebUI"**按钮，点一下浏览器会自动打开 NapCat 控制台（通常是 `http://127.0.0.1:6099`）。

输入密钥（启动器里能看到，默认是随机的）登录。

### 4.2 找到 "网络配置" → "HTTP 客户端"

> 注意是 **"HTTP 客户端"**（HTTP Client，主动 POST 给外部），**不是 "HTTP 服务器"**

### 4.3 添加一个 HTTP 客户端

| 字段 | 填什么 |
|---|---|
| **启用** | 打开 ✅ |
| **名称** | `caiji-mvp` |
| **URL** | **`https://caiji.tianku.com/onebot/event`** |
| **Token** | 留空（开发期不验签） |
| **消息格式** | `array`（推荐）或 `string` |
| **上报自身消息** | 不上报（关闭，避免循环） |

✅ **URL 是固定的，不会变**：`https://caiji.tianku.com/onebot/event`，已部署在阿里云杭州 ECS，24/7 在线，HTTPS 加密，安全可靠。

### 4.4 点"保存"

## 常见问题

### Q1: QQ 安全中心提示"非法外挂"怎么办？
A：这是 NapCat 的已知风险。**所以才让你用专用小号**——小号被封了再开一个，不影响主业务。**绝对不要用主号**。

### Q2: NapCat 重启后小号又掉了
A：QQ 9.9.26 + NapCat v4.18.1 这个组合已知很稳定。如果一直掉，可能是 QQ 检测到了 NapCat，需要重新扫码登录。后续我们可以挂"扫码自动续期"插件。

### Q3: 公网 URL 会变吗？
A：**不会**。`https://caiji.tianku.com/onebot/event` 是固定地址，阿里云服务器 24/7 在线，配一次永久用。

### Q4: 上报失败 `HeadersTimeoutError` 怎么办？
A：跟你 4-28 那条日志一样的报错。说明 NapCat 联系不上服务器——可能服务器临时故障。**截图发我，我看服务器日志立刻处理**。

### Q5: NapCat 占资源多吗？
A：NapCat 本身约 50-100 MB 内存；NTQQ 客户端约 200-500 MB。开机自启即可，不影响日常使用。
